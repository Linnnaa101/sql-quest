Schlüssel und Beziehungen

Schlüssel helfen dir zu erkennen, welche Zeilen eindeutig sind und wie Tabellen zusammengehören.

Primärschlüssel

Ein Primärschlüssel erkennt jede Zeile einer Tabelle eindeutig.

In SQL Quest ist das meistens die Spalte id.

Beispiel: kunden.id erkennt jeden Kunden eindeutig.

Fremdschlüssel

Ein Fremdschlüssel verweist auf den Primärschlüssel einer anderen Tabelle.

So entstehen Beziehungen zwischen Tabellen.

Beispiel: bestellungen.kunden_id verweist auf kunden.id.

kunden 1 → n bestellungen

Ein Kunde kann mehrere Bestellungen haben.

Eine Bestellung gehört aber genau zu einem Kunden.

Deshalb steht in der Tabelle bestellungen die Spalte kunden_id.

bestellungen 1 → n bestellpositionen

Eine Bestellung kann mehrere Bestellpositionen haben.

Eine Bestellposition gehört aber genau zu einer Bestellung.

Deshalb steht in der Tabelle bestellpositionen die Spalte bestellung_id.

produkte 1 → n bestellpositionen

Ein Produkt kann in mehreren Bestellpositionen vorkommen.

Eine Bestellposition bezieht sich aber genau auf ein Produkt.

Deshalb steht in der Tabelle bestellpositionen die Spalte produkt_id.

Beziehungen

Die Beziehungskarten zeigen immer beide Blickrichtungen: links die 1:n-Seite und im Text die n:1-Seite.

Zum Beispiel kann ein Kunde mehrere Bestellungen haben; eine Bestellung gehört umgekehrt genau zu einem Kunden.

kunden 1 ─── n bestellungen
bestellungen 1 ─── n bestellpositionen
produkte 1 ─── n bestellpositionen
Bereit zum Erstellen der Übungsdatenbank.

Schritt 3

Einweisung: Deine erste SQL-Abfrage

SQL ist eine Sprache, mit der du Daten aus Datenbanken abfragen kannst.

Eine Datenbank besteht aus Tabellen.
Eine Tabelle besteht aus Spalten und Zeilen.

In SQL Quest arbeiten wir zuerst mit der Tabelle kunden.

Mit SELECT sagst du, welche Daten du sehen möchtest.
Mit FROM sagst du, aus welcher Tabelle die Daten kommen.
Das Sternchen * bedeutet: Zeige alle Spalten.

Die erste Abfrage lautet:

SELECT * FROM kunden;

Das bedeutet:
Zeige alle Spalten aus der Tabelle kunden.
