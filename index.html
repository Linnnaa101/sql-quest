Schritt 1

Datenbank erstellen

Bevor du SQL-Abfragen schreibst, brauchst du eine Datenbank.

In SQL Quest arbeitest du mit einer kleinen Shop-Datenbank.

kunden

  • id Primärschlüssel
  • name Name
  • stadt Wohnort
  • alter_jahre Alter
  • punkte Lernpunkte

produkte

  • id Primärschlüssel
  • name Produktname
  • kategorie Warengruppe
  • preis Verkaufspreis
  • lagerbestand Vorrat

bestellungen

  • id Primärschlüssel
  • kunden_id verweist auf kunden.id
  • bestelldatum Datum
  • status Bestellstatus

bestellpositionen

  • id Primärschlüssel
  • bestellung_id verweist auf bestellungen.id
  • produkt_id verweist auf produkte.id
  • menge Anzahl
  • einzelpreis Preis pro Stück

Beziehungen

Ein Kunde kann mehrere Bestellungen haben.

Eine Bestellung kann mehrere Bestellpositionen haben.

Ein Produkt kann in mehreren Bestellpositionen vorkommen.

kunden 1 ─── n bestellungen
bestellungen 1 ─── n bestellpositionen
produkte 1 ─── n bestellpositionen
